# 인프런 강의 크롤링 & 분석

인프런(inflearn.com) 전체 강의를 Selenium으로 크롤링해 SQLite DB에 저장하고,
인기 강의 / 매출 상위 강사 등을 분석한 프로젝트.

- 크롤링 대상: https://www.inflearn.com/courses (1~142페이지)
- DB: [`../inflearn_courses.db`](../inflearn_courses.db)
- 스크립트
  - [ex05.py](ex05.py) — 크롤러 (Selenium + SQLite)
  - [ex06.py](ex06.py) — 분석 리포트 (Top 10)

## DB 스키마 (`courses` 테이블)

| 컬럼 | 설명 |
|---|---|
| `page`, `rank_in_page` | 목록 페이지 번호 / 페이지 내 순서 |
| `title`, `author` | 강의 제목 / 강사 |
| `subscriber` | 구독자수 문자열 (예: `5,800+`) |
| `rating`, `review_count` | 평점, 리뷰 개수 |
| `original_price`, `sale_price` | 정가 / 할인가 (예: `₩198,000`, `무료`) |
| `thumbnail` | 썸네일 URL |
| `url` | 강의 상세 페이지 URL (canonical, 쿼리 제거) |
| `category1`, `category2` | 대분류 / 소분류 (예: `개발 · 프로그래밍` / `풀스택`) |
| `release_date` | 출시일 추정 (가장 오래된 수강평 날짜, `YYYY-MM-DD`) |

## 크롤링 전략 (ex05.py)

1. 목록 페이지(`/courses?page_number=N`)에서 카드 단위로 기본 정보 추출
2. 카드를 감싸는 `<a>` 태그로 상세 페이지 URL 획득 (쿼리스트링 제거)
3. 무료 강의(`sale_price == "무료"`) 스킵
4. 상세 페이지 진입 후
   - 카테고리: `div.css-umsf7r.mantine-1avyp1d a[href*='/courses/']` 앵커 2개
     - NEW/BEST 뱃지가 첫 `div`라 앵커 기준으로 확실히 분리
   - 출시일: 수강평 섹션(`section.css-1h0915r.mantine-1avyp1d`) 내 "더보기" 최대 3회 클릭 후 섹션 내 날짜 중 최소값

## 프롬프트 이력 (주요)

1. 파일명 ex06.py → ex05.py 변경
2. 인기 강의 Top 10 (구독자순) / 매출 Top 10 강사 (구독자수 × 할인가 추정) 출력
3. 무료 강의 제외
4. 강의별 카테고리 수집 방법 질의
5. DB에 `url` / `category1` / `category2` / `release_date` 컬럼 추가
6. URL 저장 실패 수정 (ancestor `<a>` 탐색 + 쿼리 제거)
7. NEW/BEST 뱃지가 카테고리로 잘못 잡히는 문제 수정 (앵커 기반)
8. 수강평 섹션 정렬(최신순) + "더보기" selector 확정 → 출시일 로직 구현
9. 첫 페이지 10개 재진단 (실패 지점 파악용 diag 로깅)
10. "더보기" 3번 상한 + 그중 가장 오래된 날짜
11. 무료 제외 상태로 142페이지 전체 크롤링

## 한계 & 개선 여지

- 카테고리/출시일이 1건 누락 (재처리 필요)
- `subscriber` 근사치로 매출 추정 정밀도 제한
- `release_date`는 "가장 오래된 수강평 + 최대 3번 더보기" 기반이라 수강평이 많은 강의는 실제 출시일보다 늦은 값이 될 수 있음
- 인프런이 목록 페이지에 같은 강의를 중복 노출하는 경우가 있어 DB에 중복 행 존재 (`(title, author)` 기준 DISTINCT 필요 시 쿼리에서 처리)
